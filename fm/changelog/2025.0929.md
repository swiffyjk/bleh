# oracle
- __You can participate in the open alpha testing for ‘oracle’, a redesigned music view__
	- ‘oracle’ is a complete refresh of how album and track information is gathered and displayed to you, now powered by [MusicBrainz](https://musicbrainz.org) and community contributions via [oracle](https://github.com/katelyynn/oracle) for any fixes needed to said data.
	- The goal is to show accurate tracklists for albums - and link back to the correct albums on track pages.
	- As said previously, it is in early *alpha* and will not be the default for a while. To start using the alpha, head to the ‘Playback’ tab in settings and scroll to the bottom.
	- This is where a lot of the work went into in this release, so any feedback and suggestions is appreciated - it’ll be a long time until this is perfected.

# Profiles
- Increased the amount of recognised profile links
	- You will now see proper icons, auto-filled names, and brand colouring for said links
- Added ‘Scrobble’ as a quick access option
- Hover over the info tip to view extended features being used on a profile
	- Displays if a profile accent and/or banner is present

# Music
- __Introduced new artwork uploader__
	- Now supports a simple drag-and-drop from your Finder to Last.fm
- __You can now customise which links to display for artists, albums, and tracks__
	- Head to the ‘Interface’ settings tab and choose which services you most commonly use
- __Introduced a new title romanisation option__
	- Supported for Japanese and Korean scripts, you can now locally convert artist, album, and track titles into their romanised counterparts, e.g. メズマライザー → Mezumaraizaa
- __Right-clicking will now open a menu on grid items__
	- From here, you can quickly access the artist, photos, wiki, listeners, shouts, and tags – with additional options for expanding the artwork to it’s full size and a copy link button.
- __Added ‘viewing any track’ as a valid option for context-based colours__
	- In simpler terms, you can now have all your recent tracks be coloured based on your album artwork - as appose to only your currently scrobbling track
- Removed the Bulk Edit dependent album name in tracklist setting
	- It was causing more issues than it was solving and also never worked in the same way the original extension did. Use the in-built ‘extend track height to show album text’ option as a replacement.
- Finally updated the ‘top listeners’ preview on artist pages
- ‘Popular now’ no longer uses it’s music video for cover art
- Fixed various cases where lotus was not present
- ‘Explore in library’ is now present on search pages
- Combined artist profiles can now be fixed via lotus
- Adjusted collage defaults
