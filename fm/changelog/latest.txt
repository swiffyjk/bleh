This preference is saved between tabs. You can click points on the graph just the same as before.\n    - The date selector now also includes a handy link to the current year (2024) just in time for the end of the year.\n    - Line graphs also now have their background colour a pretty gradient and that's cool!!!\n    - There is a (probably slightly) higher chance you can get rate-limited now but in a world where using the Bulk Edit extension can load hundreds of pages that doesn't seem to be much of an issue. It's also a worthy compromise for pretty graphs i think.\n    - If you absolutely HATE these graphs and want the old one back, simply use the handy configure button present on all library pages (or head to the Music tab in settings).\n\n- __There are now additional 'insight' graphs based on the current tab__\n    - These can show information on artists, albums, and tracks on a page (you can't have all three on page, you can have two at least). Hover over each slice of the pie to see what the item in question is, unfortunately clicking into them is not available... yet (will get to this soon hopefully).\n    - No additional network requests are made for this graph to work as it utilises either the chartlist or grid already loaded :3\n    - These do not appear on the scrobbles tab!!!\n\n![New library graphs](https://katelyynn.github.io/bleh/fm/changelog/img/glacier_graphs.png)\n\n- __Header cards on regular tabs are redesigned__\n    - This redesign is the most similar to the previous design but essentially matches the rest of the recent bleh designs :3 This includes moving relavant buttons into their own group on the right with helpful text labels and icons.\n\n- __Header cards when focused on an artist, album, or track have been redesigned too__\n    - Along with the aftermentioned improvements, this panel now uses the lotus correction system. This supports the popular Bulk Edit extension out of the box so no need to worry there either.\n    - Let me know if there's other extensions people use that hook into this area and are as such no longer visible in bleh, i'm not aware of any at the moment.\n\n![Bulk edit integration](https://katelyynn.github.io/bleh/fm/changelog/img/bulk.png)\n\n- Artists in list view are now finally corrected\n- Albums and tracks in list view should now be corrected properly regardless as language\n\n- Tracks now show hover popups pretty much everywhere\n\n# Profiles\n- Recent (and) track panel configure menus now include bleh options\n    - Very handy for some people most likely (including me for testing)\n- Fixed activites sometimes displaying in a weird order\n\n![Configuration menu](https://katelyynn.github.io/bleh/fm/changelog/img/configure.png)\n\n# Header\n- Massively improved performance on the profile menu\n    - This was done by completely removing the old menu provided by Last.fm which was still causing some slowdowns despite not being visible (silly)\n\n# Other\n- Changelogs now display myself in the header\n    - Maybe other people will write changelogs someday o.O\n\n- Chart colours are now only loaded when necessary, giving a potential (but minor) performance improvement.\n- The same as above but for removing album-related page colouring when you exit a page.\n
