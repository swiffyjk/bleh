hello!! it’s been over a month since the last release, so you can guarantee this has a lot of changes – i hope everything works well :3\n\n# Collage\n- __Re-built how collages are generated__\n    - The size of a collage’s final output is now fully determined by the settings you choose – meaning you can make a 1x10 collage or a 16x9 collage or a usual square and everything works out. Do beware that currently large collages cause a *higher* file size, will most likely look into fixing this in a future release.\n- __Disabling branding and grid gaps now removes all padding__\n    - Meaning you can generate a clean grid that *only* contains your album artwork, no outer or inner black border – if that interests you.\n- The new in-built Last.fm collage feature redirects to the bleh implementation\n    - From my perspective, the bleh feature is better in every way (and open source) – so if something is missing compared to the Last.fm one, let me know or work on a contribution :3\n\n# Design\n- __Introduced new animations for grids, tracks, etc.__\n    - Beautiful satisfying staggered animations, can disable with the reduced motion setting in ‘Accessibility’.\n- __Included a new default font ‘Overpass’__\n    - This replaces the previous default font ‘Ubuntu Sans’. If the font is not satisfactory to you, you can download any font from the internet (like Google Fonts) and change your font in the bleh ‘Appearance’ tab.\n- __Unified all cards into a single card design__\n- __Redesigned track menus from the ground up__\n    - Made from scratch within bleh now!! There may be some issues that were not caught due to having to work around the annoyance of Last.fm and it’s usage of Vue (destroying element context). Should still have bulk edit functionality, and if not click the play count bar to head directly to it’s page in your library and a big bulk edit option is always available.\n- __Refactored gradient for artist plays__\n    - The transition between play counts is now actually a working gradient instead of a half-kinda working thing. The appearance of play count bars has also been adjusted too.\n- __Replaced mis-aligned buttons next to headers with links__\n    - Takes effect across your profile and music pages like your recent tracks, top artists, an album’s tracklist, etc.\n- __Aligned tracks in lists and removed individual borders__\n- Redesigned the ‘side actions’ (buttons on the right-hand side of pages)\n- If you have a profile banner active (`![banner](url to image)` in your bio), it is now also visible in the top-right profile menu, and various page banners (like settings and the Home).\n\n# Quick switcher\n- __Introduced a quick switcher__\n    - Accessible with either ⌘ or Ctrl (depending on your system) – do ⌘K ⌘Comma or ⌘Q to open. Some browsers like to forcibly hijack some of those key combinations so alternatives were added in case.\n- __Hit ⌘D to search globally__\n    - This flow allows you to enter an artist, album, track, profile, or tag to quickly navigate to where you need to be – useful for when you know the exact spelling but don’t want to waste time searching in the default way.\n- __Hit ⌘⇧K to run page-specific actions__\n    - On pages that support the functionality, you can quickly navigate to a tab with just your keyboard in this menu.\n- To accompany these shortcuts, a few extras have been added\n    - Hit ⌘P to go to your profile, ⌘S for your profile shortcut, and ⌘B for the bleh settings.\n\n# API\n- __You can now link your account with bleh’s API__\n    - The ‘bleh API’ is the Last.fm API in reality. Connecting your account in the ‘Profiles’ tab unlocks additional features – the first for now being:\n- __Integrated ‘Scrobble’ buttons on your profile and music pages__\n    - Ever been listening to a song but it doesn’t automatically scrobble? – Either try out the ‘New’ button on your profile or navigate to the page itself on Last.fm and hit ‘Scrobble’\n\n# Components\n- __Added a new date picker component__\n    - This is currently used in the new ‘Scrobble’ feature and also replaces the Library’s legacy date pickers. Do let me know if you have any trouble with these components.\n\n# Settings\n- __Introduced a new updater__\n    - Instead of being forced to update, you will be notified in the top left (next to the bleh logo) if an update has been found – or you can check for yourself in the bleh settings!!\n- __Redesigned the theme picker__\n- __Begun unifying settings together in a much nicer way__\n- Introduced a new design for toggles, range sliders, buttons, and input boxes\n- You can now view how many characters you have used in your profile bio again\n\n# Music\n- __Fixed the shout queue not working__\n    - Shouts are now ‘parsed’ (turned into fancy text) one-by-one instead of all at once which should drastically improve loading times.\n- __Refactored song tag extraction__\n    - Should now be much more optimised than before which is especially useful due to the amount of tracks you can encounter on a single page.\n- __Redesigned the edit and multiple bulk edit modals__\n    - Fully bleh designed now, with checkboxes replacing the weird toggles from before and a nicer input box design.\n- Details above listener graphs (listeners, scrobbles, etc.) are now smaller\n    - Matches the sizing in your profile library.\n- Redesigned featured items (artist’s most recent album and top track of the week)\n- __Restored top albums of recent on ‘Albums’ tab__\n    - Also fixed the sorting option disappearing at some point.\n- __Fixed being unable to sort photos on an album page__\n    - Same as above.\n- Fixed capitalisation correction for the ‘About artist’ and ‘Featured on’ panels\n\n# Profile\n- __Your recent monthly listening graph now lazy loads__\n    - Saves loading times in case you never end up scrolling down to view it.\n- Fixed follow detection breaking due to recent changes\n- Fixed the caching of profile banners being faulty\n    - It should also no longer be able to remove your own profile from the cache storage.\n- Fixed profile notes not working\n    - You can also now leave notes on users you have blocked (or if they don’t have a bio), incase you find that useful.\n\n# Seasonal\n-  __Fixed timezones with a decimal offset breaking__\n    - Seasons will now continue to function no matter what type of timezone you have active.\n\n# Markdown\n- __Extended the markdown support for legacy BBCode__\n    - When profile bios or shouts use the wiki formatting for albums (such as `[album artist=ROSÉ]rosie[/album]`) it will now display as a real clickable link. This functionality used to exist in Last.fm but was removed with the 2015 redesign.\n- Unified markdown support and link parsing together\n    - Sounds very technical but essentially everything uses the same thing now so changes will happen faster and be less prone to issues.\n\n# Other\n- __Your current device platform is now detected__\n    - This may be used more in the future for ensuring compatibility with certain features that only have significance on certain platforms – first is the ‘Emoji compatibility’ feature which is only really relevant on Windows 10 and below. This feature now automatically disables itself on macOS as it has no effect.\n- __Fixed sponsor list failing to load causing issues__\n    - Additionally, if you download a newer version of the sponsor list but are running an older version of bleh that doesn’t contain support for said badge, the badge will now display with a `?` icon.\n- Redesigned the new bookmark modal\n- When hovering over links on a profile or music wiki, a tooltip will display if the link is external\n- Fixed loading of the wiki crashing if the wiki was too short\n- Adjusted seasonal support due to new design changes\n- Added a warning if you disable style loading by mistake and want to revert
